# UI/UX 设计文档（H5）— MBTI Learn

## 1. 设计目标
- **低门槛**：首次进入 90 秒内完成“快测→开玩→结算”。
- **儿童友好**：大按钮、大字号、少输入；反馈温和但明确。
- **对话导向**：界面围绕“角色气泡 + 下一句选择”组织。
- **可扩展**：同一套组件支持多章节、多玩法模板。

## 2. 信息架构（IA）

### 2.1 主要页面
- **P0 启动/落地页**：产品一句话 + CTA（开始测试/直接开玩）
- **P1 MBTI 趣味快测**：4 题，卡片式
- **P2 类型结果页**：MBTI 卡 + 学习性格标签 + 推荐理由 + 开始第 1 关
- **P3 今日推荐/首页**：今日场景卡、继续上次、创始会员/订阅入口
- **P4 场景关卡页**：对话气泡 + 选择题/拼图题 + 点读
- **P5 结算页**：掌握句子、错题、收藏、下一关
- **P6 句子收藏/复习页（MVP 可简化）**
- **P7 订阅/创始会员页**：权益对比 + 0 元加入 + 订阅下单

### 2.2 全局元素
- 顶部：返回、进度（第几关/第几题）、静音/音量
- 底部：主操作按钮区（选项/继续/再来一题）

## 3. 关键用户流（H5）

### 3.1 新用户冷启动流
P0 → P1（快测）→ P2（结果）→ P4（第 1 关）→ P5（结算）→ P7（订阅/创始会员）→ P3（首页）

### 3.2 回访流（次日）
P3（今日推荐）→ P4（复习/新关）→ P5（结算）

## 4. 页面级设计要点

## 4.1 P1：MBTI 趣味快测
- **布局**
  - 顶部：进度条（1/4）
  - 中部：问题卡（插画 + 问题）
  - 底部：A/B 两个大按钮（至少 48px 高，便于儿童点击）
- **交互**
  - 点击即下一题（无“提交”步骤）
  - 每题可点“我不确定”（MVP 可隐藏），用于降低卡顿
- **微文案**
  - 强调“这是趣味测，会影响你看到的游戏风格”

## 4.2 P2：类型结果页
- **内容结构**
  - MBTI 卡（例如 ENFP）
  - 学习性格标签（例如“热场型/探索型”）
  - 3 条可理解的推荐理由（如“你更适合故事对话关”）
  - CTA：**立即开玩（推荐场景）**
- **注意**
  - 避免“人格定性”，用“学习偏好”表述

## 4.3 P3：今日推荐/首页
- **今日卡片**
  - 场景名（如“下次和你玩”）
  - 今日目标（学会 5 句“推迟/约时间”的回应）
  - 预计时长（2–3 分钟）
- **入口**
  - 创始会员（0 元加入）
  - 订阅（解锁全部）
  - 我的收藏（可选）

## 4.4 P4：场景关卡页（核心）

### 4.4.1 对话气泡区
- 角色气泡：Child / Mom / Dad 不同颜色
- 每句英文下方可切换显示中文（默认开，支持“一键隐藏中文”）
- 每句英文右侧：点读按钮（扬声器）

### 4.4.2 操作区（按题型切换）
- **题型 A：情境选择**
  - 显示“缺口”气泡（“你要怎么说？”）
  - 下方 2–4 个选项按钮（英文为主，中文辅助小字）
- **题型 B：下一句选择（对话拼图）**
  - 显示当前对话上下文
  - 让用户选择“下一句最合适”的选项
- **题型 C：道具卡辅助**
  - 题面可展示时间卡/规则卡（如“5 minutes”“timer”）
  - 作为提示，不是必选输入

### 4.4.3 反馈层
MVP 推荐两段式反馈（不打击、仍明确）：
- 正确：给“太好了 + 为什么这样说”
- 错误：给“更合适的是 X + 这句通常在 Y 场景用”

## 4.5 P5：结算页
- 结构：本关星级/完成度 → 今日掌握句子卡（可点读）→ 错题卡（可一键再练）
- CTA：下一关 / 复习错题 / 收藏

## 4.6 P7：订阅/创始会员页
- **信息优先级**
  1) 0 元创始会员（低门槛加入）
  2) 订阅权益（对话场景包 + 个性化推荐 + 复习）
  3) 价格与周期（待定，文案先占位）
- **合规**
  - 明确“创始会员 0 元”是否自动续费：默认不自动（建议）
  - 订阅需展示自动续费条款（若采用）

## 5. 组件规范（建议）
- **SentenceCard**
  - 英文（大）/中文（小）/点读/收藏
- **DialogueBubble**
  - 角色标签、气泡颜色、长句自动换行
- **OptionButton**
  - 英文主文案 + 中文辅助
  - 状态：默认/选中/正确/错误/禁用
- **ProgressBar**
  - 关卡进度/题目进度

## 6. 视觉规范（建议）
- 字体：系统字体（iOS/Android 默认），英文用更清晰的无衬线
- 字号：正文英文 18–22px；中文 12–14px
- 色彩：3 角色色 + 反馈色（正确绿、提示蓝、错误红但低饱和）
- 插画：轻量扁平风（家庭场景：玩具、计时器、画本）

## 7. 无障碍与儿童友好
- 点击目标 ≥ 44px
- 关键 CTA 固定底部
- 避免一次出现超过 4 个选项
- 支持“静音模式”
